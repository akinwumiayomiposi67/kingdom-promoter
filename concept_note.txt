App Concept

App type: Invitation-only PWA web app
Purpose: A transparent Christian association contribution platform where members contribute monthly, funds are tracked openly, and admins publish disbursement records with receipts.

Suggested app name ideas:
Kingdom Fund Circle (KFC)

User Roles
1. Member

A member can:

Accept invitation and register
View personal dashboard
See monthly contribution package
View virtual account number
Fund wallet
View wallet balance
See contribution history
See total monthly contribution raised by the group
View disbursement records
Download/view receipts uploaded by admin
Receive meeting notifications
View upcoming meetings
2. Admin

Admin can do everything a member can do, plus:

Invite new members
Approve/deactivate members
Create contribution packages
Set monthly contribution cycle
Create meetings
Send meeting notifications
View all member payments
View unpaid/part-paid members
Debit wallets automatically/manual confirmation
Publish monthly contribution summary
Record disbursement
Upload payment receipts
Manage supported organizations/missionaries
Export reports
Member User Journey
1. Invitation Flow
Admin enters member name, phone number, and email.
System generates a unique invitation link.
Member receives invitation by email/SMS/WhatsApp.
Member clicks the link.
Member creates account:
Full name
Email
Phone number
Password
Optional profile photo
Member accepts association terms.
Member account becomes active.

Important rule: nobody should be able to register without an invitation.

2. Onboarding Flow

After registration, member sees:

Welcome screen

Association name
Purpose of the contribution
Monthly support mission
Transparency promise
Button: “Choose My Monthly Contribution”

Member selects package:

₦10,000 monthly
₦20,000 monthly
₦50,000 monthly
₦100,000 monthly
Custom amount

After selection, system saves the member’s monthly pledge.

3. Wallet / Virtual Account Flow

Each member gets a dedicated virtual account.

Dashboard shows:

Bank name
Account number
Account name
Wallet balance
Monthly pledge amount
Contribution status

Example:

Wallet Balance: ₦25,000
Monthly Package: ₦10,000
This Month Status: Paid
Virtual Account: 1234567890 - Wema Bank

When the member transfers money into the virtual account:

Payment gateway detects payment.
Wallet is credited automatically.
Member receives notification.
System records transaction.
Dashboard updates instantly.

Recommended Nigerian providers to consider:

Paystack Dedicated Virtual Accounts
Monthly Contribution Flow
Automatic Monthly Debit

At the beginning or end of each month:

System checks each member’s selected package.
If wallet balance is enough, system debits wallet.
Contribution is marked as paid.
Member receives payment confirmation.
Amount is added to that month’s general contribution pool.

Example:

Member package: ₦10,000
Wallet balance: ₦15,000
System debits: ₦10,000
Remaining wallet balance: ₦5,000

If Wallet Balance Is Not Enough

System marks the member as:

Pending
Partially funded
Not paid

Member gets reminders:

Email
SMS
WhatsApp push notification
In-app alert

Example:

“Your March contribution of ₦10,000 is pending. Please fund your wallet using your virtual account.”

Developer Summary

Build an invitation-only PWA membership contribution platform where each member has a wallet and dedicated virtual account. Members choose a monthly contribution package, fund their wallet, and the system automatically debits monthly dues. Members can see their personal contribution history and the total group contribution for each month. Admins manage invitations, meetings, contribution cycles, disbursements, beneficiaries, and receipt uploads. The system must prioritize transparency, audit trails, notifications, and mobile-first usability.
